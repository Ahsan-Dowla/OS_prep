📚 Topics Covered

Process vs Thread Process States PCB Scheduling Context Switching

🧠 Key Learnings

  • Process vs Thread: Threads are lightweight but risky due to shared memory.
  • Process States: Most delays happen in WAITING (I/O).
  • PCB: OS stores all process info here.
  • Scheduling: Trade-off between fairness and performance.
  • Context Switching: Too many switches = performance drop.

⚔️ DevOps Insight

  • High CPU ≠ High performance
  • Most real issues = I/O bottlenecks
  • Too many threads can crash systems
  • OS is always balancing efficiency vs responsiveness

🧨 My Answers

Why too many threads slow system?
Context switching overhead + cache inefficiency + synchronization cost.

Why SJF impractical?
Cannot predict CPU burst time + starvation.

What OS optimizes?
Throughput + CPU utilization + response time.
